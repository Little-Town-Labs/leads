## Summary: Key Architectural Principles

### 1. **Single Codebase, Multiple Tenants**
- One deployment serves all tenants
- Branding & content customized per tenant
- Database queries scoped by `orgId`

### 2. **Public + Private Routes**
- **Public**: Tenant landing pages (`[tenant]/*`) - no auth
- **Private**: Dashboard (`/dashboard`) - requires Clerk auth

### 3. **Clerk Organizations = SaaS Tenants**
- Organization ID links to tenant record
- Team members join organization
- Roles control admin vs member access

### 4. **Three User Personas**
- **Potential SaaS customers**: Try demo assessment to evaluate product
- **Active SaaS customers**: Sign up, configure tenant, manage leads
- **End customers**: Anonymous visitors who take tenant's quiz

### 5. **Subdomain Routing**
- Middleware extracts subdomain
- Rewrites to `[tenant]` dynamic route
- Database lookup loads tenant config

### 6. **Dual-Purpose Assessment System**
- **Demo assessment** (`/assessment`) - Acquires SaaS customers
- **Production assessment** (`[tenant]/quiz`) - Qualifies tenant's leads
- Same infrastructure, different context and outcomes
